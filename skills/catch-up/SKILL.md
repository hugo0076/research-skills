# catch-up

Hugo has been lightly supervising and wants to re-enter the loop. Give a structured debrief.

Open with a **one-line whole-project orientation**, then go into the nitty-gritty **since the last check-in**. Do not re-summarise the whole project history.

Then walk these six. Default to using all of them (he rarely asks for a catch-up unless something has happened), but scale down if very little has:

1. What we are trying to achieve (the overall goal).
2. The sub-problem we are currently stuck on.
3. What we have been doing in that sub-problem (context for the recent work).
4. The most recent work and its results.
5. What this leans us toward, hypothesis-wise.
6. What is outstanding to fully lock it down.
